# 高校宿舍维修报修管理系统开发任务说明文档

## 一、项目名称

**基于 Spring Boot 的高校宿舍维修报修管理系统设计与实现**

------

## 二、项目目标

请开发一个基于 **Spring Boot 技术栈** 的高校宿舍维修报修管理系统。

该系统主要用于解决高校宿舍日常维修报修流程繁琐、处理进度不透明、维修记录难以统计的问题。系统需要支持学生提交报修申请，管理员审核并分配维修人员，维修人员处理工单，学生查看维修进度并进行评价。

系统需要具备完整的前后端功能，适合作为本科毕业设计项目使用。

------

## 五、核心业务流程

### 1. 学生报修流程

```
学生登录系统
↓

↓

↓
提交报修申请
↓
等待管理员审核
↓
查看报修进度
↓
```

```
↓
查看待审核报修单
↓
审核报修申请
↓
审核通过后分配维修人员
↓
查看维修进度
↓
查看维修统计数据
```

### 3. 维修人员处理流程


## 六、工单状态设计

报修工单需要有明确的状态流转。

建议状态如下：

| 状态码 | 状态名称 | 说明                             |
| ------ | -------- | -------------------------------- |
| 0      | 待审核   | 学生刚提交报修申请               |
| 1      | 审核驳回 | 管理员审核不通过                 |
| 2      | 待派单   | 管理员审核通过，等待分配维修人员 |
| 3      | 待维修   | 已分配维修人员，等待维修         |
| 4      | 维修中   | 维修人员正在处理                 |
| 5      | 已完成   | 维修人员已完成维修               |
| 6      | 已评价   | 学生已对维修结果进行评价         |
| 7      | 已取消   | 学生取消报修申请                 |

## 十三、统一返回结果要求

后端接口统一返回格式。

```
{
    "code": 200,
    "message": "操作成功",
    "data": {}
}
```

示例类：

```
@Data
public class Result<T> {
    private Integer code;
    private String message;
    private T data;

    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("操作成功");
        result.setData(data);
        return result;
    }

    public static <T> Result<T> error(String message) {
        Result<T> result = new Result<>();
        result.setCode(500);
        result.setMessage(message);
        result.setData(null);
        return result;
    }
}
```

## 十五、核心业务规则

### 1. 报修单提交规则

```
学生必须登录后才能提交报修单。
报修标题、报修类型、宿舍楼、房间、故障描述不能为空。
提交后默认状态为“待审核”。
系统自动生成唯一工单编号。
```

工单编号格式建议：

```
BX + 年月日时分秒 + 随机数
例如：BX20260509153000123
```

------

### 2. 审核规则

```
只有管理员可以审核报修单。
只有待审核状态的工单可以审核。
审核通过后，状态变为“待派单”。
审核驳回后，状态变为“审核驳回”。
审核时需要保存审核意见。
```

------

### 3. 派单规则

```
只有管理员可以派单。
只有待派单状态的工单可以派单。
只能分配给维修人员角色的用户。
派单后，状态变为“待维修”。
系统记录派单时间。
```

------

### 4. 维修处理规则

```
只有被分配的维修人员可以处理该工单。
维修人员点击开始维修后，状态变为“维修中”。
维修人员填写维修结果后，状态变为“已完成”。
系统记录开始维修时间和完成时间。
```

------

### 5. 评价规则

```
只有提交该报修单的学生可以评价。
只有已完成状态的工单可以评价。
一个工单只能评价一次。
评价后，工单状态变为“已评价”。
```

------

### 6. 取消规则

```
学生只能取消自己的工单。
只有待审核状态的工单可以取消。
取消后，状态变为“已取消”。
```
